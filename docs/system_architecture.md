# 系统架构设计文档

## 1. 系统概述

### 1.1 项目背景

本项目是一个面向NetData监控数据的智能运维问答与执行系统，旨在通过大语言模型（LLM）和多智能体（Multi-Agent）技术，实现对运维监控数据的智能分析、故障诊断和自动化操作。

### 1.2 设计目标

| 目标 | 描述 | 量化指标 |
|------|------|----------|
| 智能问答 | 基于知识库的自然语言问答 | 准确率 > 85% |
| 故障诊断 | 自动化异常检测与根因分析 | 召回率 > 90% |
| 安全执行 | 命令执行审批机制 | 风险拦截率 100% |
| 高可用 | 系统稳定运行 | 可用性 > 99.9% |
| 低延迟 | 快速响应 | P95 < 1s |

#### 3.1.2 意图分类规则

```java
public enum IntentType {
    QUERY,      // 查询类：询问系统状态、指标含义、操作方法等
    ANALYSIS,   // 诊断类：故障排查、异常分析、根因定位
    EXECUTION   // 执行类：执行命令、修改配置、重启服务等
}

// 分类规则
private static final Map<IntentType, List<Pattern>> INTENT_PATTERNS = Map.of(
    IntentType.QUERY, List.of(
        Pattern.compile("(?i)(什么是|怎么|如何|为什么|查询|查看|显示)"),
        Pattern.compile("(?i)(指标|参数|配置|状态|信息)")
    ),
    IntentType.ANALYSIS, List.of(
        Pattern.compile("(?i)(分析|诊断|排查|检查|异常|故障|问题|原因)"),
        Pattern.compile("(?i)(为什么.*高|为什么.*低|为什么.*慢)")
    ),
    IntentType.EXECUTION, List.of(
        Pattern.compile("(?i)(执行|运行|重启|停止|启动|修改|删除|创建)"),
        Pattern.compile("(?i)(帮我|请|需要).*执行")
    )
);
```

## 4. 数据模型设计

### 4.1 MySQL数据模型

```sql
-- 用户表
CREATE TABLE users (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    username VARCHAR(50) NOT NULL UNIQUE,
    email VARCHAR(100) NOT NULL UNIQUE,
    password_hash VARCHAR(255) NOT NULL,
    role VARCHAR(20) DEFAULT 'user',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);

-- 文档表
CREATE TABLE documents (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    title VARCHAR(255) NOT NULL,
    content_type VARCHAR(50),
    file_path VARCHAR(500),
    chunk_count INT DEFAULT 0,
    status VARCHAR(20) DEFAULT 'active',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 审批记录表
CREATE TABLE approvals (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    command_type VARCHAR(50) NOT NULL,
    command_content TEXT NOT NULL,
    risk_score DECIMAL(3,2) NOT NULL,
    risk_factors JSON,
    status VARCHAR(20) DEFAULT 'pending',
    requester_id BIGINT,
    approver_id BIGINT,
    executed_at TIMESTAMP,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (requester_id) REFERENCES users(id),
    FOREIGN KEY (approver_id) REFERENCES users(id)
);

-- 对话历史表
CREATE TABLE conversations (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT NOT NULL,
    title VARCHAR(255),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id)
);

CREATE TABLE messages (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    conversation_id BIGINT NOT NULL,
    role VARCHAR(20) NOT NULL,
    content TEXT NOT NULL,
    intent VARCHAR(20),
    metadata JSON,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (conversation_id) REFERENCES conversations(id)
);
```
