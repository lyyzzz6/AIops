## 主要修改内容

### 1. 开题报告修改

#### 1.1 系统定位调整
**原定位**:
> 面向NetData监控数据的自动化分析系统

**新定位**:
> 面向NetData监控数据的智能运维问答与执行系统

**修改理由**:
- 导师强调要明确"你做了啥",不能和NetData功能重复
- 新定位明确:不重做NetData的监控功能,而是在NetData数据基础上做智能问答和执行
- 对标IncidentFox项目,说明差异化

#### 1.2 技术栈明确为Python+Java混合
**原技术栈**: 全部Java(Spring Boot + Spring AI)

**新技术栈**:
- **Python部分**: PyOD + PySAD(异常检测)
- **Java部分**: Spring Boot + Spring AI(应用层、LLM集成、RAG)
- **通信方式**: REST API或消息队列

**修改理由**:
- 导师建议Python做异常检测,因为Python生态更成熟(PyOD和PySAD是业界标准)
- 导师质疑Java不适合日志分析,所以将异常检测层用Python实现
- Java保留用于应用层,因为Spring生态成熟,适合企业级应用

#### 1.3 对标IncidentFox项目
**新增内容**:
- 2.3节"相关开源项目"中新增IncidentFox介绍
- 说明IncidentFox是基于Slack的智能运维问答系统
- 本系统的差异化:
  - 基于NetData(轻量级监控)
  - 国内生态(微信/企业微信代替Slack)
  - RAG增强(运维知识库)
  - 自动执行修复命令(不仅是建议)

#### 1.4 精简内容到约20页
**删除部分**:
- 删除过长的背景意义论述
