## 1 课题背景与意义

### 1.1 研究背景

企业 IT 基础设施日益复杂，运维人员每天要处理海量监控数据。传统运维模式依赖人工经验，效率低、响应慢。Wang 等 [1] 的研究显示，传统人工运维模式下，跨域故障根因定位平均耗时超过 45 分钟。

NetData 是一款轻量级监控工具，支持 1 秒级高频数据采集，CPU 占用率低于 1%。但 NetData 仅提供数据采集和展示，缺乏智能分析和自动化处置能力。

大语言模型 (LLM) 在自然语言理解、知识推理方面的能力提升，为构建智能运维 Agent 提供了技术支持。IncidentFox 等项目证明了基于聊天的运维问答模式的可行性。

### 1.2 研究意义

本系统的研发可以直接提升运维效率：

1. **故障检测更实时**：通过 NetData 的高频采集和实时分析，秒级检测到异常。

2. **故障诊断更智能**：通过 LLM 的智能推理，自动分析故障根因。Liu 等 [6] 的研究表明，类似技术可以将平均修复时间 (MTTR) 降低 67%。

3. **故障处置更自动化**：对于标准化故障，系统可以自动生成并执行修复脚本。

4. **运维知识自动沉淀**：系统自动记录故障处理过程，沉淀为可复用的运维知识。

## 2 国内外研究现状

### 2.1 智能运维大模型研究

Wang 等 [1] 提出了面向智能运维的集成诊断、预后和决策的大模型，通过协作大 - 小模型范式，统一了信号模型和 LLM 的优势。

裴丹等 [2] 系统阐述了大语言模型时代的智能运维架构，提出了由多个 AIOps 智能代理组成的人机协同系统。

### 2.2 多智能体运维框架

Niu 等 [7] 针对人工智能数据中心的运维挑战，设计了多智能体运维框架，实现了 85.16% 的根因定位准确率。

江洁等 [3] 提出了基于大模型的多智能体 API 网关可观测监控系统，构建了五个专业智能体。

### 2.3 RAG 增强的运维系统

Wang 等 [8] 针对电力信息系统异常检测，提出了混合 RAG 增强的 LLM 多智能体系统，提高了领域知识召回率。

柴吴军等 [10] 通过"感知 - 认知 - 执行"三层架构，实现了运维合规异常从检测到处置的全程自主化。

### 2.4 相关开源项目

**IncidentFox**: 基于 Slack 的智能运维问答系统，监听 Datadog、Prometheus 等监控工具的告警，在 Slack 频道中自动回复调查进度。

**NetData**: 轻量级实时性能监控解决方案，支持多维度指标监控，超低资源占用。

**PyOD 和 PySAD**: Python 异常检测库，支持多种无监督和半监督异常检测算法，适合实时监控场景。

## 3 研究开发的基本内容

### 3.1 系统定位与边界

本系统是一个基于 NetData 监控数据的智能运维问答与执行系统。**系统不做 NetData 已做的数据采集和展示功能**,而是在 NetData 采集的数据基础上，通过 LLM 提供智能问答和自动化执行能力。

**NetData 已做的**：
- 高频数据采集 (1 秒级)
- 实时监控展示
- 基础告警通知

**本系统要做的**：
- 异常检测与根因分析 (PyOD/PySAD)
- 智能运维问答 (LLM+RAG)
- 自动化修复执行 (安全范围内)

具体来说，系统要做三件事:

1. **采集 NetData 数据**: 通过 NetData 的 API 获取监控指标数据。

2. **智能分析异常**: 使用 Python 的 PyOD/PySAD 检测异常，使用 Java+Spring AI 进行智能推理。

3. **生成执行建议**: 通过 RAG 检索运维知识库，生成可执行的运维建议或命令。

### 3.2 技术方案

#### 3.2.1 技术栈选择

根据导师的建议，本系统采用 Python+Java 混合技术栈：

**Python 部分 (异常检测层)**:
- PyOD: 用于无监督异常检测
- PySAD: 用于流式数据异常检测
- 原因:Python 生态在异常检测领域更成熟

**Java 部分 (应用层)**:
- Spring Boot 3.x: 企业级框架，微服务架构
- Spring AI: 与大模型集成，统一抽象接口
- MyBatis-Plus: 数据访问
- Milvus: 向量数据库，RAG 检索
- Neo4j: 知识图谱
- Vue.js 3 + Element Plus: 前端聊天界面
- 微信/企业微信：国内生态替代 Slack

**通信方式**:
- Python 和 Java 之间通过 REST API 或消息队列通信
- Python 处理异常检测结果，发送给 Java 层
- Java 层调用 LLM，生成运维建议

### 3.3 核心模块设计

#### 3.3.1 数据采集模块

- 通过 NetData 的 REST API 和 WebSocket 接口采集监控指标
- 采集频率为 1 秒
- 采集内容包括 CPU、内存、磁盘、网络等多维度指标
- 数据存储到 InfluxDB 时序数据库

#### 3.3.2 异常检测模块

- 使用 PyOD 进行无监督异常检测
- 使用 PySAD 进行流式数据异常检测
- 检测到异常后，将结果通过 REST API 发送给 Java 层
- 异常数据包括：异常时间、异常指标、异常值、历史基线等

**代码示例 (Python)**:

```python
from pyod.models.iforest import IForest
import numpy as np

# 加载监控数据
data = np.load('netdata_metrics.npy')

# 训练 Isolation Forest 模型
clf = IForest(contamination=0.01)
clf.fit(data)

# 检测异常
anomalies = clf.predict(data)

# 将异常结果发送给 Java 层
requests.post('http://localhost:8080/api/anomaly', 
              json={'anomalies': anomalies.tolist()})
```

#### 3.3.3 知识检索模块

- 构建运维知识库，包含历史故障案例、处理方案、最佳实践
- 使用 Milvus 向量数据库进行向量检索
- 使用 BM25 进行关键词检索
- 混合检索结果，返回最相关的知识片段

**代码示例 (Java)**:

```java
@Service
public class KnowledgeRetrievalService {
    
    @Autowired
    private MilvusService milvusService;
    
    @Autowired
    private SearchService bm25Service;
    
    public List<String> retrieve(String query) {
        // 向量检索
        List<String> vectorResults = milvusService.search(query);
        
        // 关键词检索
        List<String> keywordResults = bm25Service.search(query);
        
        // 混合检索
        return combineResults(vectorResults, keywordResults);
    }
}
```

#### 3.3.4 智能问答模块

- 使用 Spring AI 与 LLM 集成 (OpenAI 或 Ollama)
- 使用 Prompt Template 管理提示词模板
- 将异常数据、检索到的知识片段作为上下文
- 生成自然语言的运维建议

**代码示例 (Java)**:

```java
@Service
public class ChatService {
    
    @Autowired
    private ChatClient chatClient;
    
    public String generateSuggestion(AnomalyData anomaly, 
                                      List<String> knowledge) {
        String prompt = """
            你是一个运维专家。
            
            当前异常情况:
            - 异常时间：%s
            - 异常指标：%s
            - 异常值：%s
            - 历史基线：%s
            
            相关知识:
            %s
            
            请生成运维建议，包括:
            1. 可能的原因
            2. 建议的解决方案
            3. 需要执行的命令 (如果有的话)
            """.formatted(anomaly.getTime(), anomaly.getMetric(), 
                         anomaly.getValue(), anomaly.getBaseline(),
                         String.join("\n", knowledge));
        
        return chatClient.prompt(prompt)
                         .call()
                         .content();
    }
}
```

#### 3.3.5 决策执行模块

- 将运维建议转化为可执行的命令
- 支持 Linux 命令、SQL 脚本、配置修改等
- 提供安全审核机制
- 关键操作需要人工确认

**代码示例 (Java)**:

```java
@Service
public class ExecutionService {
    
    @Autowired
    private SecurityCheckService securityCheck;
    
    public ExecutionResult executeCommand(String command, 
                                          String user) {
        // 安全审核
        RiskLevel risk = securityCheck.check(command);
        
        if (risk == RiskLevel.HIGH) {
            // 高风险操作，需要人工确认
            return ExecutionResult.pendingApproval();
        }
        
        // 执行命令
        Process process = Runtime.getRuntime().exec(command);
        
        // 返回执行结果
        return ExecutionResult.success(process.getInputStream());
    }
}
```

## 4 创新点

### 4.1 NetData+LLM 的首次结合

现有研究多针对通用监控数据或特定行业的专用数据，缺乏针对 NetData 这一特定监控平台的智能 Agent 设计。本系统针对 NetData 的高频采集、超低资源占用特性，设计了专门的 Python+Java 混合架构。

### 4.2 国内生态适配

IncidentFox 基于 Slack，本系统适配国内生态 (微信/企业微信),更适合国内企业使用。

### 4.3 自动化执行

IncidentFox 只提供建议，本系统不仅可以提供建议，还能自动执行修复命令 (在安全范围内)。

### 4.4 RAG 增强

构建运维知识库，缓解大模型"幻觉"问题，提升回答准确性。
